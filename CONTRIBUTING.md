# 협업 규칙

1ST 팀 GitHub 작업 규칙. **ASM-1st 조직의 모든 저장소**에 적용된다.

## 기본 흐름: 이슈 → 브랜치 → PR

1. **이슈 먼저** — 고치거나 추가할 게 있으면 먼저 이슈를 판다.
2. **이슈별 브랜치** — 이슈 번호로 브랜치를 만든다. `main`에 직접 push하지 않는다.
3. **PR로 머지** — 작업이 끝나면 PR을 올리고, 리뷰 후 `main`에 머지한다.

## 브랜치 네이밍

`<type>/<이슈번호>-<짧은-설명>`

| type | 용도 |
|------|------|
| `feat` | 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 |
| `refactor` | 리팩터링 |
| `chore` | 설정·잡일 |

예: `feat/12-multi-persona-flow`, `fix/8-login-error`, `docs/3-readme`

## 커밋 & PR

- 커밋·PR 본문에 관련 이슈를 `#12` 형태로 참조한다.
- PR 제목은 무엇을 했는지 한 줄로. 머지 시 이슈가 닫히도록 PR 본문에 `Closes #12`.
- 리뷰어 한 명 이상 승인 후 머지 권장.
